This week I spent most of my time on learning the Flask, a micro web framework of Python. Other than that I spent sometime to watch contents, read news, and have a walk with my house owner.

Flask Learning

Reason

I have a wish to do something with computers and one of its outcome is to create a web application. For that I selected Python as my OOP language and its micro web framework Flask.

I created a basic todo web app using Flask and deployed it using Heroku, a cloud based app server last week. This app perform its primary purpose but had some bugs and need some additional features. The tutorial ( Learn Flask for Python - Full Tutorial ) I used for creating the above app is a short one and only teaches the basics.

Bugs

  1. Task column accepts empty string
  2. Update task form does not show full existing task

Additional feature

  1. Login and register feature for separate users.

Progress

So I used a new tutorial ( Python Flask Tutorial: Full-Featured Web App Part 3 - Forms and User Input ) to learn how to add that feature. Then I found that the author of this channel published a series of tutorial to learn Flask from scratch to make a full featured web app.

It clearly explains the workflow and I created another web app that performs blogging by following the tutorial. This app has the above feature which I intend to add with my todo webapp but have some other issues like deployment to virtual server.

This Flask Blog app is powered by

  • OOP language - Python
  • Web framework - Flask
  • CSS framework - Bootstrap
  • Virtual server - Heroku
  • Database - Development (SQLite) and Production (Postgresql)

Features of this flask blog webapp

  1. Anyone who has internet access can use this site to see blogposts of others.
  2. Can become a registered user.
  3. Update their accounts and reset their passwords in a secure way.
  4. Can create, update and delete their own blogposts.

Issues and its current state

This app works fine while using local development server but on Heroku it does not work. The reason is in the tutorial, the author used SQLite as the database but when you deployed it in the Heroku Postgresql is used as a database for the reasons. So I am learning to add Postgresql database to my Flask Blog app and create tables inside it.
